主要目标

  1. 正确标识哪些宠物图像是小狗图像（即使品种分类错误），哪些宠物图像不是小狗图像（无论图像是否分类正确）。
     
  2. 针对小狗图像正确判断小狗品种。
     
  3. 判断哪个 CNN 模型架构（ResNet、AlexNet 或 VGG）“最能”实现目标 1 和 2。
     
  4. 思考最能实现目标 1 和 2 所需的时间，并判断对于每个算法运行所花费的时间，是否有替代解决方案能够实现“不错”的结果。

项目概要

针对所有三个图像分类算法（例如作为命令行参数的输入算法）重复实现以下功能：

  • 记录程序的运行时间
    • 使用 Time 模块计算程序的运行时
  • 从用户那获取程序输入
    • 使用命令行参数获取用户输入
  • 创建宠物图像标签
    • 使用宠物图像文件名创建标签
    • 将宠物图像标签存储在数据结构（例如字典）中
  • 创建分类器标签并比较标签
    • 使用 Classifier 函数分类图像，并创建分类器标签
    • 将分类器标签与宠物图像标签进行比较
    • 将宠物标签、分类器标签和比较结果存储在复杂的数据结构（例如列表字典）中
  • 将标签分类为“小狗”或“非小狗”
    • 使用 dognames.txt 文件将所有标签（宠物标签和分类器标签）分类为“小狗”或“非小狗”
    • 将新的分类信息存储在复杂的数据结构（例如列表字典）中
  • 计算结果
    • 使用标签和分类结果判断算法在分类图像方面的效果如何。
  • 输出结果
